C. Sistem Urgency Otomatis

🟢 FRESH < 10 menit
→
🟡 WARNING 10–20 menit
→
🔴 URGENT > 20 menit
⚠️ Saat status berubah ke URGENT, sistem membunyikan alert audio (Web Audio API) dan menampilkan strip merah di bagian atas layar beserta nomor order urgent.

Relasi Antar Tabel

🏪 Outlet → Users
1 outlet memiliki banyak users
users.outlet_id → outlets.id
🧾 Order → Items
1 order memiliki banyak order_items
order_items.order_id → orders.id
💳 Order → Payments
1 order bisa banyak payments (split)
payments.order_id → orders.id
⚙️ Item → Modifiers
1 item bisa banyak modifier options
via order_item_modifiers bridge
⏱️ Shift → Orders
1 shift memiliki banyak orders
orders.shift_id → shifts.id
📂 Category → Items
1 kategori memiliki banyak menu items
menu_items.category_id → menu_categories.id

Method Endpoint Modul Deskripsi Request Body / Params
🔐 AUTH — /api/auth
GET /api/auth/staff/:outletId POS List kasir aktif untuk login screen Params: outletId
POST /api/auth/login POS Verifikasi PIN kasir (bcryptjs) { userId, pin }
POST /api/auth/verify-manager-pin POS Verifikasi PIN Manager untuk void/diskon besar { pin, outletId }
GET /api/auth/outlets BO List semua outlet aktif —
🍔 MENU — /api/menu
GET /api/menu/outlet/:outletId POSBO Semua kategori + item + modifier groups per outlet Params: outletId
🧾 ORDERS — /api/orders
POST /api/orders POS Buat order baru (transaksi atomik) { outletId, userId, shiftId, items[], payments[], ... }
GET /api/orders KDSBO List order dengan filter outletId + kitchenStatus Query: outletId, kitchenStatus
GET /api/orders/shift/:shiftId POS Riwayat order per shift (untuk laporan) Params: shiftId
GET /api/orders/config/:outletId POS Konfigurasi POS (service charge, payment methods) Params: outletId
PATCH /api/orders/:id/status KDS Update status order atau item (KDS selesaikan) { status } atau { itemId, itemStatus }
PUT /api/orders/:id/void POS Void order (perlu PIN Manager) { reason, voidBy }
⏱️ SHIFTS — /api/shifts
GET /api/shifts/active/:outletId POS Cek shift yang sedang aktif Params: outletId
POST /api/shifts POS Buka shift baru { outletId, userId, openingCash }
PUT /api/shifts/:id/close POS Tutup shift dan generate summary { closingCash }
GET /api/shifts/:id/summary POS Rekap lengkap shift (total, void, breakdown payment) Params: id
📊 DASHBOARD — /api/dashboard
GET /api/dashboard/kpi BO KPI harian: revenue, transaksi, avg order, diskon, metode bayar Query: outletId, date
GET /api/dashboard/recent-orders BO N transaksi terbaru dengan nama kasir Query: outletId, limit
🔄 Status Order & Transisi Lengkap STATUS FLOW

Siklus Hidup Order

🆕 pending Baru dibuat dari POS
→
👨‍🍳 preparing KDS: sedang dimasak
→
🔔 ready Siap diambil
→
✅ completed Diserahkan ke pelanggan
❌ voided Dibatalkan (butuh PIN Manager)
🚫 cancelled Dibatalkan sebelum diproses

Status Order Item (Per Item di KDS)

pending Menunggu dimasak
→
ready Item sudah siap
→
completed Diserahkan
💡 Logika Auto-Update: Saat SEMUA order_items berubah ke status ready atau completed, status ORDER utama otomatis diperbarui ke ready.
