## Schema Constraints & Relationships
The live schema uses `PRAGMA foreign_keys = ON`. The following critical constraints were verified live:
- `orders` is tightly coupled to `tenants`, `outlets`, `users`, and `shifts` via ON DELETE constraints.
- Data silos correctly exist as separate disjoint tables (`cost_bahan`, `cost_recipes`, `crm_customers`) disconnected from main tables.
